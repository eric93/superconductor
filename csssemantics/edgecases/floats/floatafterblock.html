Demonstration

The first two blocks show that, in a block flow, floats get put on a different line than a block sibling. However, floats are not considered when computing box boundaries: if a float is before a block (see the middle set of boxes), it pushes the contents of the block down and the block is considered to start at the end of the previous block (not float).

The last, third block shows that, while normal boxes in a block flow are stacked vertically, a word wrapping scheme is used for the floats.

The middle example highlights different bugs in both Mozilla/WebKit and Opera. When a float overflows into a following block, the block should try to float its contents to the right of the float. Mozilla/WebKit fail here. Opera succeeds, but considers the block to start to the right of the float, introducing a different bug.
